Keyword: test-data

test-data is a metadata field specifically for FlightPath. It tells FlightPath that you always want to run a csvpath you are developing or testing against a certain file. This saves you a couple of seconds opening a file each time you run.

To use test-data create a metadata comment above your csvpath. In it, add the field with test-data:filepath, where filepath is the file with your data.

You don't have to remember to remove the test-data metadata field when you push your csvpath to production. Only FlightPath looks for the field.

Usage

This csvpath will always run within FlightPath using the identified file when you right-click and select Run or hit ctrl-r. ~ test-data: my_csv_files/test.csv ~ $[0][ print("I'm running") ]
