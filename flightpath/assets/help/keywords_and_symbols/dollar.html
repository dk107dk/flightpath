Symbol: $

The dollar sign is the anchor of a reference. $ is used in all four types of references:

  • References that point to named-files, named-paths, or results to be used in a new run
  • print() references that display header values, runtime stats, or variables
  • References that feed data into a function
  • A whole csvpaths in the form $file[scan-instruction][match components]

Usage

A named-file reference looks like this. $invoices.files.Acme/orders/202:last
When using references in print statements you don't need the name. $. is enough. The current csvpath, named or unnamed, is clear from the context. $[*][ @hello = "World" print("Hi $.variables.hello") ]
A csvpath match component can look up a variable from a previous run or a previous csvpath in the same run.

In this case, the $people#invites part of the reference is pointing to the named-results people and within that, the csvpath identified as invites.

The most typical use of this type of reference is for a csvpath later in the run to refer to a csvpath that was earlier in the run.

$[*][ in(#first_name, $people#invites.variables.given_name ) ]
