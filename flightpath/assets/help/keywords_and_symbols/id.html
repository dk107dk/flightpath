Keyword: ID / Id / id / NAME / Name / name

When multiple csvpaths run together as a named-paths group it is very helpful to identify them individually. Their identity shows up in validation and error messages, the log, and in references, and the resulting file outputs.

Csvpaths are given identities by adding a metadata comment above (or less commonly below) the csvpath statement. Comments are wrapped in ~ characters. Within the comment metadata fields are identitied by a colon separating the name from the value. To set an identity use one of these metadata field names:

  • id
  • Id
  • ID
  • name
  • Name
  • NAME
If two are set the identity is taken in this order. E.g. an id will set the identity, overriding a name

If you do not give your csvpaths an identity the 0-based index within the named-paths group will be used. You will also see the index in the metadata resulting from a run referred to as the csvpath's identity and as the automatically-set value of the NAME metadata field, unless you set NAME yourself. Note that NAME is the lowest priority identity keyword and is overriden by the others.

Usage

This csvpath will cause a function error message that will include the csvpath's identity so you can find the problem quickly. ~ id: invoices description: this csvpath breaks because "five" is not a number. ~ $[*][ @n = add("five", no()) ]
