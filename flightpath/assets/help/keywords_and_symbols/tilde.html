Symbol: ~

The ~ surrounds a comment.

There are two types of comments:

  • Metadata comments that come above or (less often) below a csvpath
  • Matching comments that are intermixed with the match components within the matching part of a csvpath

Match component comments are simply a way to add documentation or comment out functionality you don't want to delete, but also don't want to be active.

Metadata comments are more interesting. They can have arbitrary text, whatever needed. But they can also have metadata fields. A metadata field is any word followed by a colon. The value of the field is all the text that comes before the next metadata keyword.

Metadata fields can be seen in the metadata of named-paths runs. More importantly, they are used as configuration modes. The modes control certain CsvPath Framework behavior on a csvpath-by-csvpath basis. Modes are powerful. You can read about them in their own documentation section.

The id/Id/ID or name/Name/NAME metadata field is also key. It assigns an identity to a csvpath that can help in reviewing logs, validation errors, metadata, and other places where knowing what csvpath was the source is important.

And, last but not least, integrations and other tools, including FlightPath, have their own metadata fields they look to for input from the user or an automation setup. These include:

  • test-data
  • test-delimiter
  • test-quotechar

Usage

This comment comments-out a match component that would raise an error. $[*][ print("hello world") ~ add( "five", 3) ~ ]
This comment sets the identity of this csvpath to hello world example. ~ id: hello world example ~ $[*][ print("hello world") ]
This comment adds two user-defined metadata fields:
  • description, equal to this is my first csvpath
  • color_of_the_sky, set to the value blue
~ description: this is my first csvpath color_of_the_sky: blue ~ $[*][ print("hello world")]
This comment tells FlightPath to use test.csv as the test data and the pipe char as the delimiter during development. ~ test-data:test.xlsx test-delimiter: | ~ $[*][ print("hello world") ]
