Reference type: metadata

The metadata reference type indicates values in the currently executing csvpath's metadata collection.

A csvpath's metadata collection is the set of user-defined and CsvPath Framework-defined fields found in an external comment. External comments are comments that come before, or less commonly after the csvpath expression itself. That means before the root, starting with the '$' character, or after the last ']' character that closes the matching part of the statement.

Within the external comment, a metadata field is a word followed by a colon. The modes are captured from comments to metadata. Any user-defined field is likewise captured.

The most common function of metadata is their use in print() functions. Metadata is also seen in the results of named-paths group runs in the meta.json file.

Usage

This reference prints the current header's user-defined description field. ~ description:An example of using metadata in a print function. ~ $[*][ print("This is the description: $.metadata.description") ]
