Reference type: results

The results reference type points to the files generated by past named-paths group runs. Past named-paths group runs are sometimes referred to as named-results. Results references are used to:

  • Create run references
  • Programmatically find results

Results type references allow you to pick out data files within the named-results name with great specificity. Using a results reference you can:

  • Point to any data.csv file
  • Use a dynamic combination of:
    • name
    • path
    • run datetime
    • results index

Named-file references can have the following dynamic forms:

  • $name.results.identity
  • $name.results.:today
  • $name.results.:today:last
  • $name.results.:today:last.identity
  • $name.results.:today:index
  • $name.results.YYYY-MM-DD_HH-MM-SS:last.identity
  • $name.results.path/to/results/:today:last.identity
  • $name.results.path/to/results:index
  • $name.results.path/to/results:index.identity
Where index is a integer indicating the run order and identity is the name or ID of a csvpath within the named-paths group. If a date or path is used in the third position it is partial-match capable.

A named-file reference can append these five tokens to the word following the type word results and the name following the third dot, if any:

  • :first
  • :last
  • :index
  • :yesterday
  • :today

Unlike file references, results references are not able to pull in multiple data files for iterative runs. However, programmatically, getting a list of matching results is supported.

Usage

This reference points to the first instance of a file registered under the named-file vacations that had a file name starting with flights. $vacations.files.modes/overseas/flights:first
This reference points to the same named-file vacations first file registered after March 1, 2025. Technically :all is the default, so :first is important. $vacations.files.2025-03-01_:first
