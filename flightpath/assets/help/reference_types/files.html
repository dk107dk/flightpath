Reference type: files

The files reference type points to named-files. It is used to create run references.

The files type references allow you to pick out data files within the named-files name with great specificity. In the usualcase the named-file name points to the most recent data arrived. Using a files reference you can point to:

  • Any version of any file
  • Any file name ever registered as the named-file
  • A dynamic combination of:
    • name
    • path
    • arrival time
    • version index or version time
    • globally unique version fingerprint
Technically, a fingerprint is an SHA256 hash value created from the bytes of the file.

A file reference can point to multiple files. Because it can work on a partial path, that means you can use a reference to run a named-paths group against many files; whereas, typically we run one named-paths group against one named-file.

Named-file references can have the following dynamic forms:

  • $name.files.fingerprint
  • $name.files.partial/or/full/path/to/file
  • $name.files.:index (where index is an arrival order int)
  • $name.files.partial/or/full/path/to/file:index
  • $name.files.YYYY-MM-DD_MM-HH-SS (arrival time)
  • $name.files.partial/or/full/path/to/file.YYYY-MM-DD_MM-HH-SS (version arrival time)
A named-file reference can append the following tokens to the (up to) two segments following the word files, with the exception of references that use fingerprints:
  • :all
  • :first
  • :before
  • :after
  • :last
  • :index (where index is a 0-n integer)

As with all references, named-file references are powerful and best used only when needed for rewind, replay, or disambiguation.

Usage

This reference points to the first instance of a file registered under the named-file vacations that had a file name starting with flights. $vacations.files.modes/overseas/flights:first
This reference points to the same named-file vacations first file registered after March 1, 2025. Technically :all is the default, so :first is important. $vacations.files.2025-03-01_:first
