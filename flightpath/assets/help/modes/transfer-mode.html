Mode: transfer-mode

transfer-mode let's you copy data.csv or unmatched.csv to an arbitrary location in the transfers directory. The transfers directory is configured in config/config.ini under [results] transfers.

To use transfer-mode use the form unmatched > var-name, where var-name is the name of a variable holding the relative path under the transfer directory. You may have as many transfers as you like. Add more transfers by separating them with commas.

Note that transfer-mode has no effect on the original data. In keeping with CsvPath Framework's copy-on-write semantics, your data remains immutably registered in the archive. Also keep in mind that you can achieve a similar transfer result using the SFTP or CKAN integrations.

Usage

This csvpath transfers its data.csv output to an external file named for a company.

If the transfer directory is configured in config.ini to be at /data/inbound, the transferred file of first names will land at path like /data/inbound/companies/01234567.txt.

~ id: transferring transfer-mode: data > my-path ~ $[*][ #firstname @my-path = concat("companies/", #company_id, ".txt") ]
