Mode: return-mode

return-mode is a key part of how CsvPath Framework gives you the flexibility to tailor your validation strategy to your use case. With return mode you are selecting if you want to collect matching lines or unmatched lines.

When return-mode is matches all matching lines will be returned by next() or collect(). (fast_forward() never returns lines, regardless of mode). This is the default behavior.

When return-mode is no-matches only the unmatched lines will be returned by next() or collect().

The simplicity of return-mode makes a big difference in combination with logic-mode and unmatched-mode.

return-mode Tells if we care about matching or unmatched lines.
logic-mode Specifies if a line must meet all criteria or just one criteria in order to be a match.
unmatched-mode Determines if we capture unmatched lines or just drop them.

Usage

This setup returns only the unmatched lines. In this case we are collecting only the even numbered lines.

~ return-mode:no-matches ~ $[*][ odd(line_number()) ]
