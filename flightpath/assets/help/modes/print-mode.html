Mode: print-mode

print-mode supports printing errors and user-defined messages to any number of Printer objects using the print() and error() functions.

Printers send text to print queues. By default a "standard out" printer is enabled that prints to the console, as well as to the printouts.txt a file generated by named-paths group runs. If you don't want anything printed to the console you can set no-default.

Usage

The default printer can be shut off with no-default. Other printers can remain active.

In this case, both match components will print on every line of data. print() prints to the The World printouts stream. And the custom error message goes to the custom errors printouts.

However, the built-in error you would expect from trying to add an integer to a string does not appear in the default printout because the default printout is suppressed. The error will nevertheless still be captured to errors.json if this is a named-paths group run.

~ print-mode:no-default ~ $[*][ add(5, "five") print("This message will go out to The World", "The World") error("Oh, no, we need to send an error", "custom errors") ]
