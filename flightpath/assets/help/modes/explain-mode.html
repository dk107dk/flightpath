Mode: explain-mode

The explain-mode setting configures the CsvPath Framework to provide additional information about line matching. The values are explain and no-explain. explain-mode defaults to no-explain.

When set, a step-by-step explanation of the values, assignments, match, etc. are dumped to the log on level INFO for each line in the file being processed. This can be an aid to debugging but it is performance expensive. The hit can be around 20-25%.

explain-mode is set in an external comment. Like all the modes, it is a metadata field, so it follows the name+colon form of user-set metadata.

Usage

explain means the log file will include additional information explaining how the match decision for each line was made. ~ explain-mode:explain ~ $[*][ yes() ]
The output from the above might look like this. 2025-05-22 15:37:05,300 - INFO - csvpath - Dumping explanation: 2025-05-22 15:37:05,300 - INFO - csvpath - String(string.notnone:cat) did matching resulting in True 2025-05-22 15:37:05,300 - INFO - csvpath - String(string.notnone:bert) did matching resulting in True 2025-05-22 15:37:05,300 - INFO - csvpath - String(string:miew) did matching resulting in True 2025-05-22 15:37:05,300 - INFO - csvpath - Line(line:None) did matching resulting in True
