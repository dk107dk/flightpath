Mode: error-mode

error-mode determines how errors are presented in printouts. The values are bare and full. Error mode overrides the error format setting in the CsvPath Framework config file on a csvpath-by-csvpath basis.

error-mode is set in an external comment. Like all the modes, it is a metadata field, so it follows the name+colon form of user-set metadata. The value set in config.ini will take precedence when the mode is not used.

Usage

full means format errors in printouts using the config.ini file's [error] section, format key. You can set the format for all csvpaths in all runs using the config panel in FlightPath. Or you can use this mode to set it for a specific csvpath.

In this case the output would have no information beyond the error message. More complete information would still be available in error JSON objects collected for the run.

~ error-mode:bare ~ $[*][ yes() ]
