Qualifier: increase

increase is the opposite of decrease. It enforces that a match component's value always goes up. Increase is a guard, not a validation. That means it prevents a variable from going down, but doesn't cause an error if the value you attempt to assign to the variable increases.

Availability

Functions Headers Variables
No No Yes

Examples

The value of a stays equal to the total number of lines. The value of b counts up as the number of lines seen increases. $[*][ @a.increase = minus( total_lines(), line_number() ) @b.increase = line_number() ]
